💡 문제 정의 및 해결

Problem: 기존 여행 계획 수립에 평균 30분 이상 소요되는 번거로움과 높은 진입 장벽.
Solution: LLM(Gemini)을 활용해 10가지 사용자 요구사항을 분석, 5분 이내에 정형화된 계획을 즉시 제공 (80% 시간 단축).

핵심 기능 (Key Features)

1. Pydantic을 활용한 완벽한 AI 데이터 정형화

AI의 환각(Hallucination) 및 JSON 포맷 에러를 해결하기 위해 FastAPI + Pydantic 파이프라인을 구축했습니다. 엄격한 스키마 정의 코드를 통해 Gemini API의 응답을 검증하고, Swagger 문서에 반영하여 백엔드의 신뢰성을 시각적으로 입증했습니다.

  • FastAPI Pydantic Schema 정의 (코드 레벨 검증)
  • 자동 생성된 Swagger 문서를 통한 스키마 확인
Backend Code (Models.py)
Swagger API Docs
2. 풀스택 관점의 UX 최적화 및 결과물

사용자에게 복잡한 입력을 직관적으로 받기 위해 React로 Multi-Step Form을 구현했습니다. 백엔드에서 검증된 데이터는 최종적으로 프론트엔드에서 모바일 친화적인 카드 UI로 깔끔하게 시각화되어 제공됩니다.

  • 직관적인 입력 경험 (Multi-Step Form)
  • 모바일 최적화된 결과 카드 UI
Input Form (Frontend)
Final Result UI

💭 회고 (Retrospective)

배운 점 (Learned)

1인 풀스택 개발로서 FE/BE 분리 환경의 네트워크 통신(CORS)을 깊이 이해하게 되었습니다. 특히, LLM의 비정형 데이터를 Pydantic으로 정형화하는 과정을 통해 AI 서비스를 실제 프로덕트 레벨로 끌어올리는 핵심 기술을 습득했습니다.

개선점 (Improvements)

향후 지도 API 연동(Pinning)과 데이터베이스(Firebase/SQLite) 저장을 통해 서비스의 완성도를 높일 계획입니다. 또한 AI 오류 시 자동 재시도(Retry) 로직을 추가하여 사용자 경험을 더욱 견고하게 만들 예정입니다.
